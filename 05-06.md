private void loadData() {

## **二、修复导入无 `StationUuid` 的 M3U 文件闪退**

### **问题根源**

- 从 M3U 导入的电台通常没有服务端 UUID（`StationUuid` 为空字符串或 null）
- 后续添加到 `CustomStationManager` 后，应用可能调用 `getById(station.StationUuid)` 或 `has(station.StationUuid)`，若 UUID 为 null 会抛出 `NullPointerException`
- 另外，在播放时也可能因 UUID 为空导致问题。

### **解决方案：为无 UUID 的电台自动生成唯一标识**

修改 `StationSaveManager.java` 中的 `LoadM3UReader` 方法，在创建 `DataRadioStation` 对象后，如果 `StationUuid` 为空，则生成随机 UUID。

**文件路径**：`app/src/main/java/net/programmierecke/radiodroid2/StationSaveManager.java`

找到 `LoadM3UReader` 方法中创建 `DataRadioStation` 的部分（约第 260 行附近），添加如下代码：

java

```
// 原代码
DataRadioStation station = new DataRadioStation();
station.StationUuid = stationUuid;
station.Name = stationName;
station.StreamUrl = stationUrl;
station.IconUrl = stationIconUrl;

// 新增：如果 UUID 为空，生成一个随机 UUID
if (TextUtils.isEmpty(station.StationUuid)) {
    station.StationUuid = java.util.UUID.randomUUID().toString();
    // 可选：同时设置 ChangeUuid 避免后续刷新逻辑出错
    station.ChangeUuid = station.StationUuid;
}

resultStations.add(station);
```

同时，确保 `DataRadioStation` 的 `hasValidUuid()` 方法能正确处理随机生成的 UUID（非空且符合格式），以及 `getById` 相关方法不会因 null 而崩溃。
